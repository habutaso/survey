# U3c 機能設計 — 業務ルール（計算コア assessment-core）

> 対象: U3c 計算カーネルの制約・検証・不変条件。PBT 不変条件（INV）は US-805 / NFR-04 / §7 PBT に対応。
> 確定回答: Q1–Q10 すべて A。PBT 範囲 Q9=A（範囲・決定論・単調性・区分整合・按分保存）。

---

## 1. 区分判定ルール（FR-24）

| ID | ルール |
|---|---|
| BR-1 | 被害度区分は損害割合（％）から6区分へ写像する。区間は下限包含・上限排他。 |
| BR-2 | 閾値: 全壊≥50 / 大規模半壊[40,50) / 中規模半壊[30,40) / 半壊[20,30) / 準半壊[10,20) / 一部損壊[0,10)。 |
| BR-3 | 区分判定に渡す損害割合は小数第1位に四捨五入した値（Q8=A）。 |
| BR-4 | `damageRatio` は常に [0,100] にクランプされる（負値・100超を許容しない）。 |

## 2. 第1次調査ルール（FR-20〜22 / Q10=A）

| ID | ルール |
|---|---|
| BR-25 | 外力・流失等フラグ（住家流失／地盤洗掘／基礎流失／全階天井までの浸水）のいずれか true → **全壊（100%）確定**。他入力・階按分に優先する。 |
| BR-24 | 外力非該当時、第1次損害割合 = 傾斜寄与 ＋ 浸水深寄与（既定方式・プレースホルダ）。合算後 [0,100] にクランプ。未入力寄与は加算対象外。 |
| BR-27 | 傾斜・浸水深がともに未入力（null）かつ外力非該当 → 損害割合 0（一部損壊）。 |
| BR-28 | 浸水深・傾斜の換算は単調増加（値が大きいほど損害割合が大きい）マスタ表に従う（INV-3 で検証）。 |

## 3. 第2次調査ルール（FR-23 / Q4=A）

| ID | ルール |
|---|---|
| BR-21 | 構造種別（wood/nonWood）ごとの構成比合計は 100%（マスタ不変条件）。 |
| BR-26 | `partDamages` の各部位はマスタ定義部位に含まれること。未定義部位は検証エラー（計算に混入させない）。 |
| BR-29 | 各部位損傷率は [0,100]。範囲外は検証エラー。 |
| BR-30 | 住家損害割合（按分前）= Σ(部位損傷率 × 構成比 / 100)。入力に無い部位は損傷率0として寄与0。 |

## 4. 階按分ルール（FR-28 / US-404 / Q5=A）

| ID | ルール |
|---|---|
| BR-20 | 階別床面積比（`FloorRatio.ratio`）の合計は 100%。逸脱は検証エラー（許容誤差は実装で定義、例: ±0.1）。 |
| BR-23 | 床面積比が未指定（null）または単一階 → 按分なし（単一階100%扱い、`DEFAULT_FLOOR_RATIO`）。 |
| BR-31 | 階按分後の住家損害割合 = Σ(階別損害割合 × 階床面積比 / 100)、[0,100] にクランプ。 |
| BR-32 | 各 `FloorRatio.ratio` は [0,100]、`floor` は正整数。 |

## 5. 共通・精度ルール（Q8=A）

| ID | ルール |
|---|---|
| BR-22 | すべての損害割合（中間・最終）は最終的に [0,100] にクランプ。 |
| BR-33 | 内部計算は高精度浮動小数。丸めは区分判定直前と結果格納時のみ（`roundRatio`＝小数第1位四捨五入）。 |
| BR-34 | `basis` の中間値（contribution 等）は丸め前の値を保持してよい（追跡性）。 |

## 6. マスタ・出典ルール（Q3=A）

| ID | ルール |
|---|---|
| BR-35 | マスタ数値（構成比・浸水深換算・傾斜換算）は `domain/assessment/constants` を唯一の真実の源とし、各定数に `@source`（出典／プレースホルダ表記）を付す。 |
| BR-36 | 現リリースのマスタ実数値は**代表値プレースホルダ**であり、`docs/references/`（内閣府 運用指針 水害編・記入の手引き）またはユーザー確定値で差替予定。差替時に型・ロジック・PBT は不変。 |
| BR-37 | 6区分閾値（BR-2）は FR-24 で確定済みのため、プレースホルダではなく確定値。 |

## 7. PBT 不変条件（US-805 / NFR-04 / Q9=A）

`assessment` 純粋関数に対し fast-check（@3.23.2, 既定 runs）で以下を検証する。

| ID | 不変条件 | 対象 |
|---|---|---|
| **INV-1（範囲）** | 任意の有効入力に対し `0 ≤ damageRatio ≤ 100`、`damageLevel ∈ DAMAGE_LEVEL_LIST`。 | compute*/classify/applyFloorRatio |
| **INV-2（決定論）** | 同一入力を2回評価 → 完全に同一の `AssessmentResult`（FR-27）。 | compute*/classify/applyFloorRatio |
| **INV-3（単調性）** | 他を固定し、部位損傷率↑／浸水深↑／傾斜↑ → `damageRatio` は非減少（換算表が単調なら保存）。 | computeSecond/computeFirst |
| **INV-4（区分整合）** | 任意 `r∈[0,100]` で `classifyDamageLevel(r)` が BR-2 の境界表と一致（境界値 10/20/30/40/50 を重点）。 | classifyDamageLevel |
| **INV-5（按分保存）** | 階別床面積比合計=100% かつ全階の損害割合が同一 `v` → 按分後＝`v`（恒等）。比率の並べ替えで結果不変。 | applyFloorRatio |

### 7.1 補助不変条件（必要に応じ追加）

| ID | 不変条件 |
|---|---|
| INV-6 | 外力フラグいずれか true → 必ず `totalCollapse`（BR-25）。 |
| INV-7 | 構成比合計=100% のマスタで全部位損傷率=100% → 住家損害割合=100（按分なし時）。 |
| INV-8 | `basis.partBreakdown` の `contribution` 合計 ＝ `combinedRatio`（丸め前, 第2次）。 |

### 7.2 ジェネレータ方針

- `damageRatio`/`compositionRatio`/`tiltRatio`/`inundationDepthCm`: 非負実数（上限付き）。
- `partDamages`: マスタ定義部位からの部分集合 + [0,100] 損傷率。
- `floorApportionment`: 比率合計=100 になるよう正規化したジェネレータ（fc）。
- `externalForceFlags`: 4 boolean の組合せ（全 false ケースと any-true ケースを網羅）。

## 8. エラー・検証方針

- U3c は純粋計算カーネル。**入力の構文検証（zod）は呼び出し側（U3a/U3b の controller / U-Cross バリデータ）**で完了している前提。
- ドメイン不変条件違反（構成比合計≠100、床面積比合計≠100、未定義部位、範囲外）は**ドメイン例外**（U-Cross の `ValidationError` 系）で fail-closed に扱う。これらは「マスタ不整合（開発時バグ）」と「入力不正（実行時）」を区別してメッセージ化する。
- 計算自体は例外を投げず常に [0,100] の結果を返す（クランプ）。投げるのは前提違反（マスタ/入力の構造不正）のみ。
