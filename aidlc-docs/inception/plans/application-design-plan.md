## 2. コンポーネント案（要件からの導出）
DDD ガイドライン（UseCase/Model/Store）に従い、以下のドメイン・コンポーネントを想定:
- **User/Auth/Role**: 認証（Cognito 踏襲）、ユーザー、ロール（調査員/管理者/閲覧者）、認可。
- **Survey（調査集約）**: 家屋情報・PII・GPS・状態機械（下書き→提出→承認→確定）・調査区分（第1次/第2次）・親調査参照・正式判定。
- **Assessment/被害度計算エンジン**: 第1次（外力/傾斜/浸水深）＋第2次（部位別×構成比）、階按分（2階割合）、損害割合→6区分。
- **Photo**: 撮影・S3・部位/全体紐付け。
- **Export**: PDF（調査票様式）/ CSV。
- **SurveyWizard/進捗（ステッパー）**: ガイド付きステップ進行・進捗パンくず・ステップ定義。
- **横断**: 監査ログ、認可、入力検証。

## 4. フォローアップ質問（回答分析で検出した矛盾の解消）

### Question AD2-FU: IndexedDB ローカル保存とオフライン方針の整合（必須）

AD2 のご回答（IndexedDB に入力内容・画像を保存し、調査終了後にサーバへ一括送信）は、**承認済みの要件と矛盾**します:
- **NFR-02**: 「初期リリースは常時オンライン前提。オフライン対応は将来拡張＝本リリースのスコープ外」
- **§9 スコープ外**: 「オフライン入力・同期（将来拡張）」
- **US-204**: 下書きの保存・再開（サーバ保存・別端末再開を含意）

加えて、PII（被災者情報）と画像をブラウザ IndexedDB に保存することは、セキュリティ・ベースライン（保存時暗号化・PII 保護・SECURITY-01）に関わります。

この方針を採用するには要件（NFR-02／§9）の更新が必要です。どう進めますか？

A) **オフライン/ローカルファーストを正式採用する**（NFR-02・§9 を更新してオフライン入力・同期をスコープに含める）。IndexedDB に下書き・画像を保持し、調査終了後にサーバへ一括同期。要件へ反映するため Requirements にループバックして再承認する。

B) **要件どおり常時オンラインを維持する**（AD2 を実質 A 相当に変更）。サーバに進捗・下書きを保存し、別端末再開・監査・暗号化を担保。IndexedDB はネットワーク瞬断時の一時バッファ等の軽微な用途に限定。

C) **限定的ローカル保存（オフライン非対応のまま）**: UI 進行位置と一時入力のみ IndexedDB に保持（再開補助）。**PII・画像はローカル恒久保存せず**逐次サーバ送信。NFR-02 のオンライン前提は維持。

X) Other（[Answer]: の後に記述）

[Answer]: A
オフライン/ローカルファーストを正式採用。NFR-02・§9 を更新し、Requirements にループバックして再承認する。
