# ユーザーストーリー生成計画

本ドキュメントは、要件定義書（`aidlc-docs/inception/requirements/requirements.md`）をユーザーストーリーへ変換するための計画です。下部の質問（`[Answer]:`）にご回答ください。回答に基づき `stories.md` と `personas.md` を生成します。

---

## 1. 方法論（実行チェックリスト）

- [x] ペルソナの定義（調査員／管理者／閲覧者、必要に応じ被災者）
- [x] ストーリーの分割方針の決定（下記の質問で確定）
- [x] エピック単位の整理（調査管理／入力・撮影／被害度計算／承認ワークフロー／結果出力／認証・権限／第2次調査）
- [x] 各ストーリーを INVEST 基準で記述（Independent, Negotiable, Valuable, Estimable, Small, Testable）
- [x] 各ストーリーに受け入れ基準（Acceptance Criteria）を付与
- [x] ペルソナとストーリーのマッピング
- [x] 要件 ID（FR/NFR/AC）とのトレーサビリティ付与
- [x] `stories.md` と `personas.md` の生成

## 2. ストーリー記述フォーマット（標準）

```
 US-xxx: [タイトル]
As a [ペルソナ], I want [目的], so that [価値].
受け入れ基準:
- [ ] ...
関連要件: FR-xx, ...
```

## 3. 対象エピック（要件からの導出・案）

- E1 調査管理（作成・家屋情報・GPS・状態）
- E2 入力と画像撮影（部位別入力・撮影）
- E3 被害度自動計算（第1次: 外力/傾斜/浸水深、第2次: 部位別×構成比、6区分）
- E4 承認ワークフロー（下書き→提出→承認→確定）
- E5 第2次調査・再調査（関連付け・正式判定の選択）
- E6 結果表示・出力（画面表示・PDF・CSV）
- E7 認証・ロール・権限
- E8 横断（監査ログ・セキュリティ・エラー処理）

## 5. 確定方針（回答反映）

- **分割の主軸**: ユーザージャーニー別（調査の開始→入力・撮影→提出→承認→確定→出力、加えて第2次調査/再調査ジャーニー）（Q1=C）
- **粒度**: 細かめ（1画面・1操作単位）（Q2=A）
- **受け入れ基準**: Given/When/Then 形式（Q3=B）
- **被災者**: 間接ステークホルダーとして personas に最小限記載（申請・再調査の契機）（Q4=B）
- **横断要件**: セキュリティ・監査・計算精度（PBT）は専用の横断ストーリー／制約条件として明示（Q5=A）
- **優先度**: MVP／後続を付与（Q6=A）

### ジャーニー構成（案）
- J1 ログイン・権限（認証、ロール別アクセス）
- J2 調査の開始（家屋情報登録・GPS・下書き作成）
- J3 第1次調査の入力・撮影（外力/傾斜/浸水深、部位別撮影、全体写真）
- J4 被害度の自動計算と確認（損害割合→6区分の表示）
- J5 提出・承認・確定（多段階ワークフロー）
- J6 第2次調査・再調査（第1次確定後の再調査開始、外部＋内部部位入力、正式判定の選択）
- J7 結果出力（画面表示・PDF・CSV）
- J8 横断的制約（セキュリティ・監査・計算精度・エラー処理）
