# ペルソナ (Personas)

住家被害認定調査（水害）効率化アプリの利用者・関係者の archetype。各ストーリー（`stories.md`）から参照する。

---

## P1: 調査員（Surveyor）— 主操作者
- **役割**: 被災現場でタブレットを用い、住家の被害状況を撮影・入力する。
- **特性/状況**: 屋外・被災現場での片手操作、悪条件（明るさ・濡れ・時間制約）。被害認定の実務知識を持つが、計算は手早く正確に済ませたい。
- **モチベーション**: 現場入力を効率化し、計算ミスなく被害度を確定したい。撮影と入力を並行したい。
- **権限**: 調査の作成・入力・提出（確定・承認は不可）。第1次確定後の第2次調査（再調査）の開始は可能。
- **関連ジャーニー**: J1, J2, J3, J4, J5(提出), J6(入力)

## P2: 管理者（Administrator）— 承認・確定者
- **役割**: 自治体の被害認定業務の管理者。提出された調査を承認・確定し、全調査を管理する。
- **特性/状況**: 多数の調査を捌く。正確性・監査性・公平性を重視。第1次/第2次の結果を比較し正式判定を選ぶ。
- **モチベーション**: 品質を担保しつつ滞りなく罹災証明の基礎データを確定したい。
- **権限**: 承認・確定、差戻し、全調査の管理、第2次調査開始、正式判定の選択。
- **関連ジャーニー**: J1, J5(承認・確定), J6(開始・正式判定), J7

## P3: 閲覧者（Viewer）— 参照のみ
- **役割**: 自治体担当者など。調査・結果を参照する（編集不可）。
- **特性/状況**: 状況把握・問い合わせ対応のために情報を参照。
- **モチベーション**: 最新の調査状況・判定結果を確認したい。
- **権限**: 参照のみ。
- **関連ジャーニー**: J1, J7（参照）

## P4: 被災者（Resident）— 間接ステークホルダー（非操作者）
- **役割**: 被害認定の対象者。アプリは直接操作しない。
- **関与**: 罹災証明の申請者であり、第1次結果への不服・申請が第2次調査（再調査）の契機となる。
- **モチベーション**: 適正な被害度判定と迅速な罹災証明の交付。
- **備考**: 本アプリではデータ対象（PII）・再調査契機として扱い、UI 操作者ではない。
