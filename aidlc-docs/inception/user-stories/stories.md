# ユーザーストーリー (User Stories)

住家被害認定調査（水害）効率化アプリのユーザーストーリー。ユーザージャーニー別に整理し、INVEST 基準・Given/When/Then の受け入れ基準・優先度（MVP／後続）・要件トレーサビリティを付与する。ペルソナは `personas.md` を参照。

凡例: 優先度 = **MVP**（初期リリース必須）／**後続**（次フェーズ）。

---

## J1: ログイン・権限

### US-101 ログイン（MVP）
As a 調査員/管理者/閲覧者, I want アプリにログインする, so that 自分の権限に応じた機能を利用できる。
関連要件: FR-40
- **Given** 有効なアカウントを持つ利用者, **When** 認証情報でログインする, **Then** 自分のロールでアプリにアクセスできる。
- **Given** 無効な認証情報, **When** ログインを試みる, **Then** 拒否され、内部情報を含まない一般的なエラーが表示される。

### US-102 ロール別アクセス制御（MVP）
As a 利用者, I want ロールに応じて利用可能な操作が制限される, so that 不正な操作や情報漏えいを防げる。
関連要件: FR-41, FR-42, FR-43
- **Given** 閲覧者でログイン中, **When** 調査の編集・承認操作を試みる, **Then** サーバ側で拒否される（UI 非表示に依存しない）。
- **Given** 調査員でログイン中, **When** 承認・確定操作を試みる, **Then** 拒否される。

### US-103 ログアウト（MVP）
As a 利用者, I want ログアウトする, so that セッションを安全に終了できる。
関連要件: FR-40, SECURITY-12
- **Given** ログイン中, **When** ログアウトする, **Then** セッションが無効化され、保護ページにアクセスできない。

## J2: 調査の開始

### US-201 第1次調査の新規作成（MVP）
As a 調査員, I want 1家屋を単位とする第1次調査を新規作成する, so that 現地調査を開始できる。
関連要件: FR-01, FR-06
- **Given** 調査員でログイン中, **When** 家屋識別情報（住所・家屋番号・構造種別等）を入力して作成する, **Then** 調査区分=第1次・状態=下書きの調査が作成される。
- **Given** 必須の家屋情報が未入力, **When** 作成しようとする, **Then** 入力検証エラーが表示され作成されない。

### US-202 被災者情報（PII）の登録（MVP）
As a 調査員, I want 被災者の氏名・連絡先等を調査に登録する, so that 罹災証明の基礎情報として紐付けできる。
関連要件: FR-02, SECURITY-01
- **Given** 調査の編集中, **When** 被災者情報を入力・保存する, **Then** 暗号化保存され、権限のない利用者には表示されない。

### US-203 調査位置（GPS）の記録（MVP）
As a 調査員, I want 調査地点の位置情報を記録する, so that 現地を特定できる。
関連要件: FR-03
- **Given** 位置情報が利用可能, **When** 調査を作成/撮影する, **Then** 緯度経度が記録される。
- **Given** 位置情報が取得不可, **When** 続行する, **Then** 位置なしでも調査は継続でき、その旨が記録される。

### US-204 下書きの保存・再開（ローカルファースト）（MVP）
As a 調査員, I want 入力途中の調査をローカル（IndexedDB）に下書き保存する, so that オフラインでも中断・再開できる。
関連要件: FR-04, FR-18
- **Given** 入力途中, **When** 下書き保存する, **Then** 入力内容・画像が端末ローカル（IndexedDB）に保持され、後で再開できる。
- **Given** オフラインでアプリを再読込/端末スリープ復帰, **When** 調査を開く, **Then** ローカルの下書きが復元される。

### US-207 オフライン入力と提出時同期（MVP）
As a 調査員, I want 通信不通の現場でも入力・撮影を続け、提出時にサーバへ一括同期する, so that 電波状況に左右されず現場作業を完了できる。
関連要件: FR-18, FR-19, NFR-02, SECURITY-01
- **Given** オフライン状態, **When** 入力・撮影を行う, **Then** すべて IndexedDB に保持され作業を継続できる。
- **Given** ローカルに下書きがある, **When** 「提出」を実行する, **Then** 入力内容・画像がサーバへ一括送信され、状態=提出に遷移する。
- **Given** 提出時に通信不通/送信失敗, **When** 同期を試みる, **Then** ローカルデータを保持してキューイングし、オンライン復帰後に再試行できる。
- **Given** 同期が成功確認された, **When** 処理が完了する, **Then** ローカルの PII・画像が確実に消去される（成功確認前は消去しない）。

### US-205 ガイド付きステップ進行・進捗パンくず（MVP）
As a 調査員, I want 調査入力がステップで案内され、進捗パンくず（ステッパー）で現在地と残作業が見える, so that 今何をしていて後何が残っているかを把握できる。
関連要件: FR-15, FR-16, FR-17
- **Given** 調査入力中, **When** 画面を見る, **Then** 標準順序（住民情報入力→1次/2次選択→調査入力→（該当時）2階部分の割合調整→計算結果確認）が表示され、現在・完了・残ステップが識別できる。
- **Given** 単層住家など該当しないステップがある, **When** 進行する, **Then** 当該ステップはスキップ表示される。

### US-206 ステップ間の移動（MVP）
As a 調査員, I want 完了済みステップへ戻って内容を見直す, so that 入力を修正・確認できる。
関連要件: FR-16, FR-15
- **Given** 複数ステップを進行済み, **When** パンくずの既存ステップを選ぶ, **Then** 入力済みデータを保持したまま当該ステップへ移動できる。
- **Given** 前提未充足の先のステップ, **When** 移動を試みる, **Then** 進めず必要な入力が示される。

## J3: 第1次調査の入力・撮影

### US-301 外力・流失該当の入力（MVP）
As a 調査員, I want 外力・流失等の該当有無を入力する, so that 全壊該当を判定できる。
関連要件: FR-20
- **Given** 第1次調査の入力中, **When** 「住家流失・地盤洗掘・基礎流失・全部の階の天井までの浸水」等の該当を選択する, **Then** 損害割合算定で全壊として扱われる。

### US-302 傾斜の実測値入力（MVP）
As a 調査員, I want 外壁・柱の傾斜の実測値を入力する, so that 傾斜による損害割合を反映できる。
関連要件: FR-21, FR-11
- **Given** 第1次調査の入力中, **When** 傾斜の実測値を数値入力する, **Then** 範囲・型が検証され、損害割合算定に反映される。

### US-303 浸水深の実測値入力（MVP）
As a 調査員, I want 床上/床下の浸水深（cm）を入力する, so that 浸水深による損害割合を反映できる。
関連要件: FR-22, FR-11
- **Given** 第1次調査の入力中, **When** 浸水深を数値入力する, **Then** 範囲・型が検証され、損害割合算定に反映される。

### US-304 部位ごとの画像撮影・添付（MVP）
As a 調査員, I want 各部位の写真を撮影・添付する（1部位に複数枚可）, so that 判定根拠を残せる。
関連要件: FR-12, FR-14
- **Given** 入力フローの部位ステップ, **When** 撮影/選択して添付する, **Then** 画像が当該部位に紐付き、複数枚を保持できる。
- **Given** 撮影と入力を並行, **When** 部位を切り替える, **Then** 入力済みデータと画像が保持される。

### US-305 全体写真の撮影・添付（MVP）
As a 調査員, I want 調査全体の写真を撮影・添付する, so that 住家全体の状況を記録できる。
関連要件: FR-13
- **Given** 調査の入力中, **When** 全体写真を撮影/添付する, **Then** 調査全体に紐付いて保持される。

## J4: 被害度の自動計算と確認

### US-401 第1次の損害割合 自動計算（MVP）
As a 調査員, I want 入力値から損害割合（％）が自動計算される, so that 手計算の手間とミスをなくせる。
関連要件: FR-20, FR-21, FR-22, FR-25, FR-27, NFR-04
- **Given** 外力/傾斜/浸水深の入力が揃っている, **When** 計算を実行する, **Then** 運用指針準拠の損害割合（0–100%）が算出される。
- **Given** 同一の入力, **When** 再計算する, **Then** 同一の結果になる（決定論的）。

### US-402 被害度6区分の判定・表示（MVP）
As a 調査員/管理者, I want 損害割合から6区分の被害度が判定・表示される, so that 家屋が何段階の被害かを把握できる。
関連要件: FR-24, FR-26, FR-30
- **Given** 損害割合が算出済み, **When** 判定する, **Then** 6区分（全壊/大規模半壊/中規模半壊/半壊/準半壊/準半壊に至らない）のいずれかに排他的にマッピングされる。
- **Given** 損害割合がちょうど境界値（例 50%/40%/30%/20%/10%）, **When** 判定する, **Then** 運用指針どおりの区分になる。

### US-403 計算根拠の確認（後続）
As a 管理者, I want 経路別（外力/傾斜/浸水深/部位別）の中間値を確認する, so that 判定の妥当性を検証できる。
関連要件: FR-26
- **Given** 計算済みの調査, **When** 根拠を表示する, **Then** 各経路の寄与・中間値が確認できる。

### US-404 2階部分の割合調整（MVP）
As a 調査員, I want 複数階建ての住家で階ごとの床面積比率（例: 1階・2階の按分）を調整する, so that 階構成に応じた損害割合を正しく算定できる。
関連要件: FR-28, FR-17
- **Given** 複数階建ての調査入力中, **When** 階ごとの床面積比率を調整する, **Then** 当該按分が部位別損害割合の算定に反映され、計算結果が変化する。
- **Given** 単層住家, **When** 進行する, **Then** 当該ステップはスキップされる。
- **Given** 比率の合計が不正（合計≠100% 等）, **When** 確定しようとする, **Then** 検証エラーになる。

## J5: 提出・承認・確定

### US-501 調査員による提出（MVP）
As a 調査員, I want 入力完了した調査を提出する, so that 管理者の承認に回せる。
関連要件: FR-04, FR-42
- **Given** 下書きの調査, **When** 提出する, **Then** 状態=提出に遷移し、以降 調査員は内容を編集できない（差戻しまで）。

### US-502 管理者による差戻し（後続）
As a 管理者, I want 提出された調査を差戻す, so that 不備を修正させられる。
関連要件: FR-04
- **Given** 提出状態の調査, **When** 差戻す, **Then** 状態=下書きに戻り、調査員が再編集できる。

### US-503 管理者による承認（MVP）
As a 管理者, I want 提出された調査を承認する, so that 確定に進められる。
関連要件: FR-04, FR-42
- **Given** 提出状態の調査, **When** 管理者が承認する, **Then** 状態=承認に遷移する。
- **Given** 調査員/閲覧者, **When** 承認を試みる, **Then** 拒否される。

### US-504 管理者による確定（MVP）
As a 管理者, I want 承認済みの調査を確定する, so that 被害度を正式に確定できる。
関連要件: FR-04, FR-05, FR-42
- **Given** 承認状態の調査, **When** 確定する, **Then** 状態=確定に遷移し、実施者・日時が監査記録される。

### US-505 確定後の編集不可（MVP）
As a システム, I want 確定済み調査の変更を禁止する, so that 判定結果の信頼性を担保できる。
関連要件: FR-05
- **Given** 確定済みの調査, **When** 誰かが編集を試みる, **Then** 拒否される（再調査は別記録で対応）。

## J6: 第2次調査・再調査

### US-601 第2次調査（再調査）の開始（MVP）
As a 管理者または調査員, I want 確定済みの第1次調査に対し第2次調査を開始する, so that 詳細な再判定を行える。
関連要件: FR-07, FR-08
- **Given** 第1次調査が確定済み, **When** 第2次調査を開始する, **Then** 第1次に関連付いた調査区分=第2次の新規記録が作成される。
- **Given** 第1次が未確定, **When** 第2次開始を試みる, **Then** 拒否される。

### US-602 外部部位の損傷率入力（MVP）
As a 調査員, I want 外部部位（屋根・基礎・外壁）の損傷率を入力する, so that 部位別算定に反映できる。
関連要件: FR-23, FR-11
- **Given** 第2次調査の入力中, **When** 外部部位の損傷率を数値入力する, **Then** 範囲（0–100%）・型が検証され保持される。

### US-603 内部部位の損傷率入力（MVP）
As a 調査員, I want 内部部位（天井・床・内壁・建具・設備 等）の損傷率を入力する, so that 内部立入判定を反映できる。
関連要件: FR-23, FR-11
- **Given** 第2次調査の入力中, **When** 内部部位の損傷率を数値入力する, **Then** 範囲・型が検証され保持される。

### US-604 第2次の損害割合・6区分 自動計算（MVP）
As a 調査員/管理者, I want 部位別損傷率×構成比で損害割合・6区分が自動計算される, so that 詳細判定結果を得られる。
関連要件: FR-23, FR-24, FR-25, FR-27, NFR-04
- **Given** 外部・内部の全部位の損傷率が入力済み, **When** 計算する, **Then** 構成比で加重合計した損害割合と6区分が算出される。
- **Given** 全部位の損傷率=100%, **When** 計算する, **Then** 住家全体=100%（構成比総和=1 と整合）。

### US-605 第1次/第2次結果の併記表示（MVP）
As a 管理者/閲覧者, I want 同一家屋の第1次・第2次の結果を併記で確認する, so that 比較できる。
関連要件: FR-09
- **Given** 第1次・第2次の双方に結果がある, **When** 家屋の結果を表示する, **Then** 両方の損害割合・区分が併記される。

### US-606 正式判定の選択・確定（MVP）
As a 管理者, I want 第1次/第2次のいずれを正式とするか選択・確定する, so that 正式な被害度を一つに定められる。
関連要件: FR-09
- **Given** 双方の結果が併存, **When** 管理者が正式判定を選択し確定する, **Then** 採用区分・選択者・日時が記録され、正式判定として表示される。
- **Given** 調査員/閲覧者, **When** 正式判定の選択を試みる, **Then** 拒否される。

## J7: 結果表示・出力

### US-701 結果の PDF 出力（MVP）
As a 調査員/管理者, I want 調査結果を PDF 出力する, so that 調査票・判定結果を記録・共有できる。
関連要件: FR-31, FR-33
- **Given** 計算済みの調査, **When** PDF を出力する, **Then** 調査票項目（運用指針様式準拠）と損害割合・被害度区分を含む PDF が生成される。

### US-702 複数調査の CSV エクスポート（MVP）
As a 管理者, I want 複数調査の結果を CSV/Excel でエクスポートする, so that 集計・他システム連携に使える。
関連要件: FR-32
- **Given** 複数の調査, **When** 期間/条件でエクスポートする, **Then** 該当調査の主要項目・判定結果を含む CSV が出力される。

### US-703 調査一覧・検索（MVP）
As a 利用者, I want 調査を一覧・検索する, so that 目的の調査に素早く到達できる。
関連要件: NFR-03, FR-43
- **Given** 多数の調査, **When** 住所・状態・区分等で検索する, **Then** 権限範囲内の調査が応答性よく一覧表示される。

## J8: 横断的制約（専用ストーリー）

### US-801 入力値の検証（MVP）
As a システム, I want すべての数値・文字入力を範囲・型・形式で検証する, so that 不正値・インジェクションを防ぎ計算の正確性を保てる。
関連要件: FR-11, SECURITY-05, NFR-04
- **Given** 任意の API 入力, **When** 受信する, **Then** zod スキーマで型・範囲・最大長を検証し、不正は拒否される。
- **Given** 損傷率/浸水深に範囲外値, **When** 入力する, **Then** 拒否され明示的なエラーになる。

### US-802 リソース認可（オブジェクト/機能レベル）（MVP）
As a システム, I want すべての調査アクセスを所有権・ロールで認可する, so that 他者データ閲覧（IDOR）や権限外操作を防げる。
関連要件: FR-43, SECURITY-08
- **Given** 他者の調査 ID, **When** 取得/編集を試みる, **Then** 権限がなければ拒否される。
- **Given** 管理者専用操作, **When** 非管理者が呼ぶ, **Then** サーバ側で拒否される。

### US-803 監査ログ（MVP）
As a 管理者, I want 状態遷移・PII/確定データの変更を監査記録する, so that 説明責任・追跡性を確保できる。
関連要件: NFR-08, SECURITY-13, SECURITY-14
- **Given** 提出/承認/確定/正式判定/PII 変更, **When** 操作が行われる, **Then** 実施者・日時（・前後値）が改ざん困難な形で記録される。
- **Given** 認証/認可失敗, **When** 発生する, **Then** アラート対象として記録される。

### US-804 保存時・通信時の暗号化（MVP）
As a システム, I want データを保存時・通信時に暗号化する, so that 個人情報を保護できる。
関連要件: SECURITY-01, NFR-05
- **Given** DB/S3 への保存, **When** データを書き込む, **Then** 保存時暗号化が有効である。
- **Given** クライアント-サーバ通信, **When** データを送受信する, **Then** TLS で保護される。

### US-805 計算ロジックの正確性検証（MVP）
As a 開発/品質担当, I want 計算ロジックを PBT と例示ベースで検証する, so that 運用指針準拠の正確性を担保できる。
関連要件: NFR-04, AC-7, AC-2
- **Given** 計算関数, **When** PBT を実行する, **Then** 損害割合 0–100%・区分の境界/排他/網羅・構成比総和整合・決定論性などの不変条件が保たれる。
- **Given** 代表シナリオ, **When** 例示ベーステストを実行する, **Then** 期待値どおりの区分になる。

### US-806 セキュアな既定動作（MVP）
As a システム, I want セキュリティヘッダ・エラー秘匿・フェイルクローズを既定とする, so that 設定ミス由来の脆弱性を防げる。
関連要件: SECURITY-04, SECURITY-09, SECURITY-15
- **Given** HTML 配信エンドポイント, **When** 応答する, **Then** CSP 等のセキュリティヘッダが付与される。
- **Given** サーバ内部エラー, **When** 発生する, **Then** 利用者には一般的なメッセージのみ返し、内部詳細を秘匿し、処理は安全側に倒れる（fail closed）。
- **Given** デモ用 Task/User, **When** リリース構成を確認する, **Then** 既に削除されている。
